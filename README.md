## コンセプト
思い出は、残すものではなく、みんなで思い出すもの。


## 対象ユーザ
思い出はあるのに、みんなで振り返る機会を失っている人たち


## 利用の流れ
### 思い出を残すとき
1. NFCチップ入りのカードをかざしてアプリを起動
2. 思い出のタイトルを入力
3. 集合写真を撮る
4. 限られた枚数の写真を撮影
5. 写真とメッセージを保存
### 思い出を振り返るとき
1. NFCチップ入りのカードをかざしてアプリを起動
2. トップには集まるメンバーの集合写真と思い出のタイトルがある
3. 参加者それぞれがカードを持ち寄る
4. カードの裏を組み合わせて「アルバムを開くカギ」を作る
5. 全員がそろったときだけ、アルバムが開く
6. 同じ画面を囲みながら、思い出を振り返る


## 推しポイント
- ひとりでは開けないアルバム  
 → みんなが集まらないと思い出せない設計

- 写真の量をあえて制限  
 → 1枚1枚に意味が残り、記憶に残る

- カードというモノの存在  
 → スマホの中に埋もれない、思い出の入口になる

- 感情と空気を思い出す体験  
 → 写真を見るだけでなく、その場の会話が自然に生まれる

## 開発体制

### 役割分担
- 安達涼太：フロントエンド
- 笹木颯太：バックエンド
- 高畠大地：動画、UX、発表流れ
- 門脇知香：UIUX、発表資料
- 江戸大晟：カードデザイン

### 開発における工夫した点
#### プロジェクトマネジメント・チーム開発の工夫
- GitHub IssuesとFigJamを併用し、実装に必要なタスクの洗い出しと進捗管理を行った
- タスクを可視化することで、担当や進行状況の認識ズレを防止した
- Discordにて毎日19時〜23時頃に定例会兼作業通話を実施し、作業中の疑問をすぐに解決でき、開発の停滞を防げた
- 全員が集まれない日は、日報チャンネルを活用して進捗を共有した
#### 技術的な工夫
- FigmaのDev Modeを活用し、Material UIベースのUI設計を実施
- 各メンバーが機能やツールを使いこなせるよう、習熟したメンバーから指導を受けながら習得した
#### スケジュール管理の工夫
- MVPを設定し、必要な機能を整理した上で逆算的にタスク期限を設計
- 開発初期の約2日間をアイデア出しに充て、「使ってみたいと思える体験」を意識した機能設計につながった
